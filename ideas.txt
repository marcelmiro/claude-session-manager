CSM Ideas Backlog
=================

## Actions & Workflows
- c → open worktree folder in Cursor IDE (cursor {path})
- d → launch dev server in worktree (configurable command per repo)
- w → worktree manager sub-view (create/delete/list worktrees with session counts)
- X → kill session + optionally delete worktree
- / → filter/search sessions by repo, branch, or name
- Tab → cycle sort mode (status / modified / repo / context)

## Session Grouping & Display
- Toggle grouping on/off
- Session tagging / custom labels
- Configurable columns (show/hide context, lines, etc.)
- Show lines modified per session

## Worktree Management
- Safety checks before worktree deletion (uncommitted changes, running sessions)
- Show session count per worktree

## Cleanup & Maintenance
- Detect dead sessions (process gone, pane still open or closed)
- Offer to clean up orphaned session files
- Detect zombie tmux panes (no process, showing error/exit code)

## Configuration
- Configurable refresh interval
- Per-repo dev commands (bun --hot vs bun run dev vs custom)
- Repo allowlist/blocklist
- Custom keybindings
- Worktree path template

## Quality of Life
- Quick-send keystrokes to a session without switching (approve tool call from manager)
- Session duration tracking (how long has this been running?)
- Show model being used per session (opus vs sonnet)
- Copy session ID to clipboard
- Export session log to markdown
- Multiple tmux session support (not just "default")
